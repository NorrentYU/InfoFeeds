# Task Contract: Telegram Test

## 任务目标
验证 Telegram 模块是否满足数据有效性、格式一致性和可用性要求。

## 输入（固定）
1. 测试源：
   - `https://t.me/cookiesreads`
   - `https://t.me/web3list`
2. 时间窗口：最近 24h。
3. 目标输出字段：
   - `source_type/source_name/title/content/url/published_at/fetched_at`

## 测试用例（必须全部执行）
1. 窗口过滤：仅返回窗口内数据。
2. 抓取稳定性：同参数重复执行结果一致（允许条数微小波动但结构一致）。
3. 外链解析：能正确提取原文链接。
4. 正文有效性：`content` 非空且非拦截页文本。
5. 降级路径：无外链但正文有效时，允许 `url` 为空，且必须伴随 `no_external_link` 失败记录。
6. 链接有效性：非降级记录的 `url` 为原文链接，不是 Telegram 消息链接。
7. 字段完整性：输出字段完整且类型正确。
8. 模板兼容性：结果可正确写入“信息集会”的 Telegram 分节。

## 失败处理
1. 任何 P0/P1 失败直接打回开发修复。
2. 若命中 CAPTCHA/防火墙，记录证据并判为“无效正文”，不得算通过。

## 交付物
1. 测试报告（测试时间、窗口、样例链接、结果）。
2. 缺陷列表（分级+复现步骤）。
3. 回归结果（修复后重测结论）。

## 通过判定
1. 全部用例通过。
2. 无未关闭 P0/P1 缺陷。
3. 报告证据齐全且可复查。
