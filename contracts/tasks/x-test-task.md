# Task Contract: X Test

## 任务目标
验证 X 模块是否满足“CDP 主路径 + 多级兜底”的 For You 抓取稳定性、有效性与格式一致性要求。

## 输入（固定）
1. `.env` 账号凭证（只读；仅兜底路径使用）。
2. 测试模式：不校验时间窗口，仅抓取 `For You` 前 5 条。
3. CDP 连接端点（默认 `http://127.0.0.1:9222`）。
4. 会话状态存储（本地持久化，支持复用）。
5. 目标输出字段：
   - `source_type/source_name/title/content/url/published_at/fetched_at`
6. 浏览器环境：独立专用 profile（与用户日常 Chrome 隔离）。

## 测试用例（必须全部执行）
1. CDP 可用性：`npm run x:cdp-smoke` 可连通并产出 `reports/x-cdp-smoke.json`。
2. CDP 不可用失败：关闭/未开启端口时返回明确 `cdp_unavailable` 或同级明确错误，不得误报为凭证错误。
3. CDP 未登录失败：端口可连但未登录时返回 `cdp_not_logged_in` 或同级明确错误，不得输出伪记录。
4. 登录可用性：可进入首页并成功定位 `For You` 标签。
5. 流类型校验：结果来自 `For You`，不是 `Following`。
6. 条数校验：成功返回 5 条记录（若阻塞需提供证据）。
7. 去重校验：5 条记录链接唯一。
8. 链接有效性：`url` 可为外链原文链接或推文状态链接，但必须是可用原始链接，不能是跳转壳。
9. 文本有效性：`content` 非空且非登录页/挑战页文案。
10. 字段完整性：输出字段完整且类型正确。
11. 模板兼容性：结果可正确写入“信息集会”的 `X` 分节。
12. 会话复用校验：连续两次执行，第二次不应触发密码登录流程。
13. 人工接管校验：挑战触发后可在受控专用 profile 中完成人工验证并继续抓取。
14. 新鲜度校验：若首轮内容超过 24h，系统会自动刷新重试（最多 3 次）并给出最终状态。

## 失败处理
1. 任何 P0/P1 失败直接打回开发修复。
2. 若出现验证码/安全验证/风控挑战，必须附证据并标记阻塞类型。

## 交付物
1. 测试报告（测试时间、抓取条数、样例链接、结果）。
2. 缺陷列表（分级+复现步骤）。
3. 回归结果（修复后重测结论）。
4. `reports/x-cdp-smoke.json` 快照或引用路径。

## 通过判定
1. 全部用例通过。
2. 无未关闭 P0/P1 缺陷。
3. 报告证据齐全且可复查。
