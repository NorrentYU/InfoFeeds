# X Module Execution Pack

## 1. 模块目标
通过非官方浏览器自动化路径，抓取用户个人 X 账号 `For You` 栏目前 20 条个性化内容，输出统一中间结构供摘要层使用。认证主路径为 CDP 直连“专用且已登录”的 Chrome。

## 2. 模块范围
1. 本轮只做 X 渠道，不触发 Others 开发。
2. 数据来源固定为登录态下 `For You` 流，不读取 `sourceList.md` 的 X 列表。
3. 生产抓取数量：前 20 条。
4. 测试抓取数量：前 5 条。
5. 认证策略（优先级固定）：
   - P1：CDP 直连专用已登录 Chrome（主路径）
   - P2：本地会话复用（session/profile）
   - P3：密码登录 + 同会话人工接管挑战（兜底）
6. 浏览器策略：必须使用独立专用 profile（禁止复用用户日常 Chrome profile）。

## 3. 开发单（给开发 agent）
开发 agent 执行文件：`contracts/tasks/x-dev-task.md`

### 3.1 输入
1. `.env` 中的 X 账号凭证（只读使用，不输出；仅兜底路径使用）。
2. 本地会话状态存储（storageState/userDataDir，持久化复用）。
3. 调度参数（默认 UTC+8 08:30）。
4. 抓取数量参数：`production=20`，`test=5`。
5. 用户摘要 prompt（由设置层注入）。
6. CDP 端点：`X_CDP_ENDPOINT`（默认 `http://127.0.0.1:9222`）。

### 3.2 实现任务
1. 先执行 CDP 连通与登录态校验（可复用 `npm run x:cdp-smoke`）：
   - 能连接 `X_CDP_ENDPOINT`
   - 能确认当前流为 `For You`
2. CDP 校验成功后，按主抓取流程获取前 N 条推文卡片（N=生产20/测试5）。
3. 若 CDP 不可用或未登录，降级到本地会话复用路径；仍失败再进入密码/人工接管兜底。
4. 人工接管必须在受控专用 profile 中完成挑战（Arkose/CAPTCHA/安全验证），完成后保存会话状态供后续复用。
5. 去重并输出统一中间结构：
   - `source_type=x`
   - `source_name=for_you`
   - `title`（可为空）
   - `content`（优先外链/引用外链正文，兜底推文文本）
   - `url`（优先外链原文链接，兜底推文状态链接）
   - `published_at`
   - `fetched_at`
6. 失败留痕：记录 CDP 连接失败、登录挑战、风控校验、页面结构异常、内容提取失败等错误类型（脱敏）。
7. 新鲜度策略：若抓取结果显示 For You 内容整体超过 24h，先执行自动刷新重试（最多 3 次）再判定阻塞。

### 3.3 禁止事项
1. 不得打印、回传、持久化明文账号密码。
2. 不得把 `Following` 或搜索结果当作 `For You` 结果。
3. 不得把跳转壳链接当作最终 `url`。
4. 不得绕过统一字段契约。
5. 不得在定时任务中反复执行“账号密码直登”。
6. 不得复用用户日常 Chrome profile（多账号多标签环境）。
7. 不得在 CDP 主路径可用时跳过该路径直接改走密码直登。

### 3.4 交付物
1. X 渠道标准化数据样例（测试 5 条 + 生产 20 条各 1 组）。
2. 开发自测记录（CDP 主路径成功 + 会话复用或人工接管至少 1 条 + 至少 1 条失败留痕）。
3. CDP 烟雾产物：`reports/x-cdp-smoke.json`。
4. 已知限制列表（若有）。

### 3.5 操作员 Runbook（CDP 主路径）
1. 启动专用 Chrome（示例，macOS）：
```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222 \
  --user-data-dir="$HOME/.infofeeds-x-profile"
```
2. 在该专用 Chrome 手动登录 X，并停留在 `https://x.com/home`。
3. 执行 `npm run x:cdp-smoke`，确认生成 `reports/x-cdp-smoke.json` 且记录数=5（测试口径）。

## 4. 测试单（给测试 agent）
测试 agent 执行文件：`contracts/tasks/x-test-task.md`

### 4.1 通用用例
1. 测试时不校验时间窗口；只验证当前 `For You` 前 5 条。
2. 抓取结果条数正确且去重生效。
3. 输出字段完整且符合统一结构。
4. 失败项不会污染成功输出。
5. 会话复用模式下不触发重复登录。

### 4.2 X 专项用例
1. 明确验证当前流是 `For You`，不是 `Following`。
2. 5 条记录链接必须有效且唯一：允许“外链原文链接”或“推文状态链接”。
3. `content` 非空且不是登录页/风控页文案。
4. CDP 端口未开启时应返回明确失败（如 `ECONNREFUSED`），不得误报登录失败。
5. CDP 已连接但未登录时应返回明确失败（未确认 `For You`），不得输出伪记录。
6. 遇到挑战（验证码/安全验证）时，必须进入人工接管流程或标记阻塞，不得伪造结果。
7. 人工接管成功后，后续一次抓取可直接复用会话完成。
8. 若首轮内容超过 24h，自动刷新重试后应能恢复或明确标记 `stale_feed`。

### 4.3 通过标准
1. 所有用例通过。
2. 无 P0/P1 未关闭缺陷。
3. 有完整测试证据可追溯。

## 5. 验收闸门（架构师）
1. 开发交付物齐全且符合契约。
2. 测试报告通过且证据完整。
3. X 结果可被下游摘要与聚合层直接消费。
4. 仅在上述 1-3 全部满足时，允许切换到 Others 模块。

## 6. 阻塞升级
1. CDP 连续连接失败（含 `ECONNREFUSED`）：升级架构师。
2. 登录连续失败或触发安全挑战：升级架构师。
3. 页面结构变动导致无法稳定定位卡片：升级架构师。
4. 契约字段不一致：先冻结新契约，再继续开发。
5. 人工接管流程不可用（无法保存会话或无法恢复抓取）：升级架构师。
