# YouTube Module Execution Pack

## 1. 模块目标
在 YouTube 渠道抓取窗口内视频更新，仅提取官方字幕/自动字幕（字幕-only），输出可用于摘要的统一中间结构。

## 2. 模块范围
1. 本轮只做 YouTube 渠道，不触发 X/Others 开发。
2. 固定测试源：
   - `https://www.youtube.com/@PeterYangYT`
   - `https://www.youtube.com/@Messari`
3. 默认窗口（YouTube 专项）：`T-48h ~ T-24h`。
4. 处理对象包含：普通视频 + 直播回放（若已可访问）。

## 3. 开发单（给开发 agent）
开发 agent 执行文件：`contracts/tasks/youtube-dev-task.md`

### 3.1 输入
1. `sourceList.md` 的 YouTube 分组。
2. 调度参数（默认 UTC+8 08:30）。
3. YouTube 专项窗口参数（默认 `T-48h ~ T-24h`）。
4. 用户摘要 prompt（由设置层注入）。

### 3.2 实现任务
1. 抓取频道视频列表并按窗口筛选。
2. 提取可用字幕（人工字幕或自动字幕）。
3. 直播回放仅在有可用字幕时纳入输出。
4. 对无字幕视频标记 `transcript_missing` 并记录原因。
5. 记录文本来源：`caption`。
6. 输出统一中间结构：
   - `source_type=youtube`
   - `source_name`
   - `title`
   - `content`（字幕文本）
   - `url`（视频原链接）
   - `published_at`
   - `fetched_at`
7. 失败留痕：记录字幕缺失、视频不可达等错误类型（脱敏）。

### 3.3 禁止事项
1. 不得把频道页链接写入最终 `url`。
2. 不得把“无字幕”视频当作成功 transcript 记录。
3. 不得阻塞整批输出；单条失败应可降级并留痕。
4. 不得绕过统一字段契约。

### 3.4 交付物
1. YouTube 渠道标准化数据样例（至少覆盖两个测试源）。
2. 开发自测记录（字幕可用样例 + 无字幕缺失样例各至少 1 条）。
3. 已知限制列表（若有）。

## 4. 测试单（给测试 agent）
测试 agent 执行文件：`contracts/tasks/youtube-test-task.md`

### 4.1 通用用例
1. 测试时不校验时间窗口；每个测试源只取最新视频进行 transcript 提取。
2. 视频链接抽取正确，`url` 指向具体视频。
3. 输出字段完整且符合统一结构。
4. 失败项不会污染成功输出。

### 4.2 YouTube 专项用例
1. 两个测试源均返回“最新视频 transcript”结果（共 2 条）。
2. 字幕提取路径可用（人工或自动字幕均可）。
3. `content` 非空且不是错误文案/占位文本。
4. 结果能按目标模板落入 `Youtube` 分节。

### 4.3 通过标准
1. 所有用例通过。
2. 无 P0/P1 未关闭缺陷。
3. 有完整测试证据可追溯。

## 5. 验收闸门（架构师）
1. 开发交付物齐全且符合契约。
2. 测试报告通过且证据完整。
3. YouTube 结果可被下游摘要与聚合层直接消费。
4. 仅在上述 1-3 全部满足时，允许切换到 X 模块。

## 6. 阻塞升级
1. 大面积字幕缺失导致有效覆盖率不足：升级架构师。
2. 大面积视频不可达（地区/权限/年龄限制）：升级架构师。
3. 契约字段不一致：先冻结新契约，再继续开发。
