# Failure Handling Rules

## 1. 失败分级
1. P0：核心链路中断（无法抓取、无法生成输出、数据严重错误）。
2. P1：主要功能异常（部分源失败、摘要不可用、格式不合规）。
3. P2：可降级问题（个别字段缺失、非关键告警）。

## 2. 排障顺序
1. 先定位失败阶段：抓取、解析、正文提取、摘要、聚合、PDF。
2. 再确认失败类型：网络失败、反爬拦截、结构变更、逻辑缺陷。
3. 最后做最小修复并立即回归。

## 3. Telegram 常见故障处理
1. 频道可访问但无更新：保留窗口证据并重试相邻窗口。
2. 消息有外链但正文抓取失败：记录外链 URL、失败类型、重试次数。
3. 命中 CAPTCHA/防火墙：标记为“无效正文”，禁止摘要入库。
4. 外链跳转多层：需解析至最终原文链接再提取正文。
5. 消息无外链但正文有效：允许降级入库（`url` 置空），并记录 `no_external_link` 证据。

## 4. Substack 常见故障处理
1. `/feed` 返回 404/403：记录源地址与响应码，重试后升级。
2. RSS 结构异常：记录解析失败字段并回退到最小可读字段集。
3. `pubDate` 缺失或异常：该条目标记无效并记录原因。
4. 内容为空或仅订阅引导文案：标记为“无效正文”，禁止进入摘要层。
5. 同文多版本重复：按 `guid` 或 `link + published_at` 去重并记录命中策略。

## 5. YouTube 常见故障处理
1. 字幕提取失败：记录字幕轨状态并标记 `transcript_missing`。
2. 直播回放尚未生成可用字幕：标记“待后处理”并顺延下一轮。
3. 视频不可达（地区/权限/年龄限制）：记录失败类型并保留证据。
4. 文本为空或质量过低：判为无效正文，不进入摘要层。

## 6. X 常见故障处理
1. 登录失败或凭证异常：记录失败类型并立即停止重试，等待人工确认。
2. CDP 端口不可达（如 `ECONNREFUSED`）：标记 `cdp_unavailable`，附端点信息并上报。
3. CDP 已连接但无可用上下文：标记 `cdp_context_missing`，要求操作员检查专用 Chrome 启动参数。
4. CDP 已连接但未登录 For You：标记 `cdp_not_logged_in`，要求在专用 Chrome 手动登录后重试。
5. 命中验证码/安全验证/风控挑战：标记阻塞并上报，不得伪造通过。
6. `For You` 标签定位失败：记录页面状态并重试；连续失败后升级。
7. 抓取条数不足（<5 或 <20）：记录缺口与滚动证据后重试。
8. 链接提取为跳转壳/聚合壳：判为无效记录并回归修复（外链原文链接与推文状态链接均可接受）。
9. 会话失效：进入人工接管登录流程，成功后保存新会话并重试抓取。
10. 人工接管超时或失败：标记 `X专项阻塞`，不中断其他渠道执行。
11. 内容整体超过 24h：执行自动刷新重试（最多 3 次）；仍旧过期则标记 `stale_feed`。

## 7. Others（newsletter）常见故障处理
1. `/feed` 返回 404/403：记录源地址与响应码，重试后升级。
2. RSS 结构异常：记录解析失败字段并回退到最小可读字段集。
3. `pubDate` 缺失或异常：该条目标记无效并记录原因。
4. 内容为空或仅订阅引导文案：标记为“无效正文”，禁止进入摘要层。
5. 同文多版本重复：按 `guid` 或 `link + published_at` 去重并记录命中策略。

## 8. 聚合层常见故障处理
1. 摘要模型超时或调用失败：按单条重试（最多 2 次），仍失败则记入 manifest。
2. 摘要格式不合规（非 `**标题** + 段落`）：标记 `summary_format_invalid` 并重试。
3. 去重误杀：记录冲突 URL 与被保留策略，回归后复核样例。
4. markdown 与 PDF 内容不一致：标记 `render_mismatch` 并阻塞发布。
5. manifest 缺失关键统计字段：标记 `manifest_invalid` 并阻塞发布。

## 9. 失败记录模板（必填）
1. 模块与来源
2. 发生时间（含时区）
3. 输入参数（窗口、源）
4. 错误现象与日志摘要（脱敏）
5. 根因判断
6. 修复动作
7. 回归结果

## 10. 升级规则
1. 同一问题连续 2 次回归失败，升级给架构师处理。
2. 涉及接口契约变更时，必须先冻结新契约再继续编码。
