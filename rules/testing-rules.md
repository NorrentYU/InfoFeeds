# Testing Rules

## 1. 测试流程
1. 先写测试计划，再执行测试，再记录证据。
2. 每次修复缺陷后必须执行回归测试，不允许只测修复点。
3. 测试记录必须包含：时间窗口、测试源、输入、输出、结论。

## 2. 测试分层
1. 单元测试：解析、去重、时间过滤、格式化等纯逻辑。
2. 集成测试：渠道抓取 -> 正文提取 -> 摘要输入准备。
3. 端到端测试：渠道结果 -> markdown 聚合 -> PDF 输出。

## 3. Telegram 渠道专项口径
1. 固定测试源：`https://t.me/cookiesreads`、`https://t.me/web3list`。
2. 测试窗口默认最近 24h。
3. 每个源至少验证 1 条有效摘要输出（外链记录优先；无外链降级需附留痕）。
4. 必须验证外链正文确实被读取，不是只总结 Telegram 消息壳。
5. 若外链被 CAPTCHA/防火墙拦截，必须判失败并记录证据，不得当作有效摘要。
6. 必须验证无外链降级路径：`url` 为空且存在 `no_external_link` 失败记录。

## 4. Substack 渠道专项口径
1. 固定测试源：`https://www.systematiclongshort.com/`、`https://www.astralcodexten.com/`。
2. 测试入口固定为源地址的 `/feed`。
3. 每个源至少验证 1 条有效“摘要输入 + 原链接”输出（或有明确无更新证据）。
4. 必须验证 `url` 为文章原文链接，不是 feed 链接。
5. 若内容为空或为订阅墙/异常页文案，必须判为无效内容。

## 5. YouTube 渠道专项口径
1. 固定测试源：`https://www.youtube.com/@PeterYangYT`、`https://www.youtube.com/@Messari`。
2. 测试时不校验时间窗口；每个测试源仅取最新视频。
3. 仅验证字幕提取路径；不执行本地转写流程。
4. 必须验证 `url` 为具体视频链接。
5. 若无字幕，必须返回 `transcript_missing` 记录；若文本无效，必须判失败并记录证据。

## 6. X 渠道专项口径
1. 测试入口固定为登录态下 `For You` 流，不依赖 `sourceList.md`。
2. 先执行 `npm run x:cdp-smoke` 作为闸门；通过后再做集成抓取验证。
3. 测试时不校验时间窗口，仅抓取并验证前 5 条。
4. 必须验证当前流为 `For You`，不是 `Following`。
5. 必须验证 5 条记录链接唯一，且均为有效原始链接（外链原文或推文状态链接）。
6. 必须覆盖 CDP 异常场景：端口不可达（如 `ECONNREFUSED`）与已连通但未登录，且失败类型明确。
7. 若出现验证码/安全验证/风控挑战，必须判阻塞并记录证据。
8. 必须验证会话复用：连续执行时第二次不应触发密码登录。
9. 必须验证人工接管：在受控专用 profile 中完成人工挑战后可继续抓取。
10. 必须验证独立 profile：抓取流程不依赖用户日常浏览器状态。
11. 必须验证 stale feed 恢复：内容超过 24h 时自动刷新重试后给出恢复或阻塞结论。

## 7. Others（newsletter）渠道专项口径
1. 固定测试源：`https://every.to/chain-of-thought/`、`https://every.to/napkin-math/`。
2. 测试入口固定为源地址的 `/feed`。
3. 每个源至少验证 1 条有效“摘要输入 + 原链接”输出（或有明确无更新证据）。
4. 必须验证 `url` 为原文链接，不是 feed 链接。
5. 若内容为空或为订阅墙/异常页文案，必须判为无效内容。

## 8. 聚合层专项口径
1. 固定执行 `contracts/tasks/aggregation-test-task.md`，不得跳过 manifest 校验。
2. 必须验证跨渠道同 URL 去重生效，最终 markdown 不重复。
3. 必须验证摘要格式为 `**标题**` + 段落正文，且不出现列表符号。
4. 必须验证默认 prompt 与“默认+用户追加 prompt”两种场景都可出报。
5. 必须验证摘要失败项被记录到 manifest，且不会污染成功摘要输出。
6. 必须验证 `digest-*.md` 与 `digest-*.pdf` 内容一致。

## 9. 输出校验
1. markdown 必须匹配既定模板，字段不可缺失。
2. 原链接必须指向原文页面，而不是平台消息壳链接或中转链接。
3. PDF 渲染后需人工抽检标题、分节、链接与摘要是否完整。

## 10. 通过判定
1. 既定测试用例全部通过。
2. 无 P0/P1 缺陷遗留。
3. 测试报告已归档并可追溯。
